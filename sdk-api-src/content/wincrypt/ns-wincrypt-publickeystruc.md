# PUBLICKEYSTRUC structure


## -description


The <b>PUBLICKEYSTRUC</b> structure, also known as the <b>BLOBHEADER</b> structure, indicates a key's <a href="https://msdn.microsoft.com/2e570727-7da0-4e17-bf5d-6fe0e6aef65b">BLOB</a> type and the algorithm that the key uses. One of these structures is located at the beginning of the <b>pbData</b> member of every <a href="https://msdn.microsoft.com/f17042c3-ba1a-408f-af55-5f171b0dee33">key BLOB</a>.

This structure is not limited to the key BLOBs generated by the PROV_RSA_BASE and PROV_RSA_SIG provider types. The <b>pbData</b> member of any new key BLOB type must begin with this structure.

### -field bVersion

Contains the version number of the key BLOB format. For example, if the BLOB is a <a href="https://msdn.microsoft.com/d007cbb9-b547-4dc7-bc22-b526f650f7c2">Digital Signature Standard</a> (DSS) version 3 key, this member will contain 3. The minimum value for this member is defined by the <b>CUR_BLOB_VERSION</b> (2) identifier.


### -field reserved

This member is reserved for future use and must be set to zero.


### -field aiKeyAlg

Contains one of the <a href="https://msdn.microsoft.com/557436b4-f7f1-4708-acc7-c6b47e6322ad">ALG_ID</a> values that identifies the algorithm of the key contained by the key BLOB.

Not all algorithm identifiers are valid with all BLOB types. For example, since an RC4 key is a session key, it cannot be exported into a PUBLICKEYBLOB.

PLAINTEXTBLOBs can be used with any algorithm or type of key combination supported by the CSP in use. Note that a 3DES key cannot be imported when the Microsoft Base Provider is in use.
